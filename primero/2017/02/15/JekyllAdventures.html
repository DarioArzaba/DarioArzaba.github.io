Jekyll Adventures

  • Primero
  • 15 Feb 2017
  • About 1 minute

Gracias a un curso que habia tomado en CodeAcademy aprendi a crear sitios en Github Pages, sin embargo no fue hasta el dia de hoy que me he decidido por incorporar Jekyll (Generador de Sitios Estaticos).

Con Jekyll podemos transformar documentos de texto en Markdown y transformarlos en HTML, es decir en un sitio web estatico; generando a su vez el layout del sitio con un renderizador Liquid.

De las multiples plantillas y estilos que existen me habia decidido por uno minimalista enfocado en texto denominado Vanilla Bean Creme Theme, el cual es una buena plantilla, sin embargo en retrospectiva deberia de haber creado mi propia plantilla desde cero. Ahora estoy utilizando una variante de Ricardo Lopez.

En Windows para poder desarrollar en Jekyll se tiene que instalar Chocolatey, Ruby, RubyGems y RubyDevKit, generalmente desde PowerShell:

PowerShell.exe -ExecutionPolicy Undefined
iwr https://chocolatey.org/install.ps1 -UseBasicParsing | iex
choco install ruby -y
ruby dk.rb init
ruby dk.rb review
ruby dk.rb install
gem install jekyll bundler
gem install jekyll-paginate-v2
gem install jekyll-feed
gem install jekyll-gist
gem install rouge
gem install wdm
cinst -Source "https://go.microsoft.com/fwlink/?LinkID=230477" libxml2
cinst -Source "https://go.microsoft.com/fwlink/?LinkID=230477" libxslt
cinst -Source "https://go.microsoft.com/fwlink/?LinkID=230477" libiconv
gem install nokogiri
jekyll new blogname
*Enter (cd) the blogname directory*
bundle exec jekyll serve

Listo, a comenzar a escribir!
