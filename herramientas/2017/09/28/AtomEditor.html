Atom, el editor de texto hackeable del siglo 21.

Atom es un editor de texto open source, multiplataforma, modular y hackeable creado en HTML, CSS (Less) y JavaScript (CoffeScript, Node). Diseñado por Github utilizando el Framework Electron sobre el Chromium Engine .

Instalación

Para instalar en Windows primero se descarga AtomSetup.exe desde el sitio principal o desde el repositorio en Github. El instalador añade los comandos atom y apm al PATH del entorno. Atom es el programa principal y Apm es el administrador de paquetes (Atom Package Manager).

Aplication Folder: %LOCALAPPDATA%\atom
Configuration Folder: %USERPROFILE%\.atom
Dev Source and Config: %USERPROFILE%/github/atom
Cache Folder: %APPDATA%\Atom
Registro: HKEY_CLASSES_ROOT\Directory\Background\shell
Registro: HKEY_CLASSES_ROOT\Directory\shell
Registro: HKEY_CLASSES_ROOT*\shell

El programa se puede utilizar de forma portable al extraer el archivo extraible (Zip o Tar) y agregar un directorio en el mismo nivel denominado .atom. La variable que controla la ubicacion de este directorio, por defecto %USERPROFILE%, es ATOM_HOME. Opcionalmente se recomienda agregar el switch --portable al ejecutable.

Si se desea instalar desde el codigo fuente se necesita Node, Python y Visual Studio con Visual C++ y Windows SDK. Opcionalmente la variable GYP_MSVS_VERSION debe corresponder a la version de Visual Studio.

cd C:\
git clone https://github.com/atom/atom.git
cd atom
script\build --create-windows-installer

La opcion del ultimo comando genera los mismos instaladores que el sitio principal. La carpeta Bin del codigo fuente debe ser agregada a PATH para poder utilizar atm correctamente. Comandos del script:

  • –code-sign: Firma la aplicacion con el certificado de Github.
  • –compress-artifacts: Genera la version portable.
  • –install[=dir]: Instala la aplicacion en ${dir}\Atom\app-dev en donde Div es por defecto %LOCALAPPDATA%.

En caso de que el firewall produzca errores de instalacion SSL, utilizar el comando apm config set strict-ssl false. En el caso de que el proxy produzca errores de HTTPS, utilizar el comando apm config set https-proxy (Proxy Adress). Probar la configuracion del proxy con apm config get https-proxy.

Para instalar paquetes de forma manual es necesario clonar su repositorio, extraer en %USERPROFILE%\.atom y ejecutar el comando atm install (Nombre) o npm install el nombre se refiere al nombre del paquete y puede ser opcional.

El programa incluye varias herramientas como un buscador de simbolos. Si se quieren buscar simbolos en el proyecto se puede instalar la utilidad Ctags Utility para crear un archivo tags que configure la generacion de tags simbolicos dentro del editor. Para mayor configuracion editar el archivo ctags dentro de %USERPROILE%\.ctags.

Herramientas

Con el uso de Snipets podemos aumentar la productividad, estos comandos son autocompletados con la tecla tab y nos permiten escribir secciones de codigo rapidamente. Para definir Snippets personalizados primero se define el Scope del paquete de cada lenguaje, los siguientes niveles definen el nombre, descripcion, abreviacion y contenido. La posicion del mouse se puede personalizar con $1, numerando cada posicion siguiente.

'.source.js':
  'console.log':
    'prefix': 'log'
    'body': 'console.log(${1:"crash"});$2'

En el ejemplo anterior el cursor tendra seleccionada la informacion entre llaves. El body se puede encerrar entre triples comillas """ si se compone de varias lineas, mientras que para escapar caracteres se utilizan dobles diagonales \\.

Cabe mencionar que desde 1.13 fueron eliminados del DOM los elementos Shadow que incluian a: ::shadow, /deep/ y :host. Esto tambien provoco que en los style fuera necesario el prefijo .syntax para todo elemento gramatical.

Atom viene integrado con herramientas para trabajar en Git y Github. En los paneles de Git y Github podemos accesar a los repositorios locales y remotos de un proyecto. Los siguientes comandos basicos nos permiten crear un repositorio remoto o uno local.

# Crear repositorio local y vincularlo a uno remoto.
git init
git clone https://github.com/NAME/repo.git
(o git clone https://username:password@github.com/NAME/repo.git)
(o git clone git clone username@host:/Name/repo.git)

# Crear un repositorio remoto y vincularlo al local.
git remote add origin https://github.com/NAME/repo.git
git push origin master

Paquetes

Al ejecutar atom.exe se carga primero el Script init.coffee que se encuentra dentro de %USERPROFILE%\.atom y que nos permite ejecutar CoffeeScript o Javascript personalizado (cambiando la extension de coffee a js).

#Ejecutar un sonido cada vez que se ejecute atom.
atom.beep()
#Convertir el texto seleccionado y en el clipboard a un link de Markdown.
atom.commands.add 'atom-text-editor', 'markdown:paste-as-link', ->
    return unless editor = atom.workspace.getActiveTextEditor()
    selection = editor.getLastSelection()
    clipboardText = atom.clipboard.read()
    selection.insertText("[#{selection.getText()}](#{clipboardText})")

El comando Create Package nos permite crear un nuevo paquete de Atom. Este paquete estara constituido por keymaps, lib, menus, spec, styles y un archivo package.json. El ultimo archivo contiene el metadata del paquete, es decir su nombre, modulo principal, version, descripcion, repositorio, licencias, engines, dependencias, activationCommands y activationHooks.

El modulo principal controla la logica y todas las vistas o paneles del paquete. Este archivo fundamental tiene acceso por defecto a los siguientes metodos:

  • Activate(state): Ejecutado al inicio del paquete. Si implementa serialize() puede recibir el estado de la ultima ventana. Realiza el trabajo de inicializacion como crear elementos DOM o ligar eventos con otro metodos.
  • Initialize(state): Ejecutado antes de la creacion del espacio activo, es decir despues de su deserializacion. Garantiza la ejecucion de codigo antes de la creacion del DOM.
  • Serialize(): Ejecutado al inicio de cerrar la ventana. Genera un codigo JSON que representa el estado de la ventana y de ser activa entrega automaticamente su informacion a Activate o Initialize cuando la ventana vuelva a ser abierta.
  • Deactivate(): Ejecutado al cerrar la ventana. Desactiva todo enlace con recursos externos, todo lo del paquete que no dependa de informacion externa es desactivado automaticamente por Atom.

Los estilos que modifique el paquete se incluyen en forma de Less dentro de la carpeta de Styles. Seran iniciados justo al terminar la carga del DOM. Los estilos de todos los componentes vienen predefinidos por Atom y por el tema actual, accesibles bajo el Styleguide. Se recomienda mantener dicho estilo.

La combinacion de teclas que active algun comando del paquete se agrega dentro de la carpeta de Keymaps. Los menus incorporan la activacion de los comandos del paquete gracias a un menu contexual o a uno en la barra de herramientas.

Flowork de Ejecucion para un paquete:

  1. Atom Inicia
  2. Abre Paquete(s)
  3. Ejecuta sus Package.json
  4. Carga Keymaps, Menus, Styles y Main Module
  5. Termina de Cargar Paquetes
  6. Usuario ejecuta comando del paquete.
  7. Ejecutar metodo Activate del Paquete
  8. Ejecutar metodo con el comando del usuario.
  9. Usuario ejecuta otro comando
  10. Ejecutar metodo con el otro comando del usuario.
  11. Ventana del paquete es cerrada. Ejecutar Serializacion.

Si se tienen problemas con Atom se puede empezar por iniciar el programa en modo seguro con atom --safe. De esta manera podemos descartar que sea un problema con los paquetes oficiales. Podemos despues ir modificando el Script init.coffee o ir desactivando los paquetes comunitarios y los temas.

Si ninguno de estos pasos resuelve el problema podemos intentar borrar el cache (window state) de la aplicacion atom --clear-window-state. Finalmente se puede intentar cambiar el nombre o borrar la carpeta de configuracion .atom.
